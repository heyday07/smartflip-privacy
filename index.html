개인정보처리방침

SmartFlip (매일매일 상식퀴즈) | 시행일: 2026년 5월 7일 | 버전 1.1

RedMoon Studio(이하 "회사")는 「개인정보 보호법」 제30조에 따라 이용자의 개인정보를 보호하고 이와 관련한 고충을 신속하고 원활하게 처리하기 위하여 다음과 같이 개인정보처리방침을 수립·공개합니다.

한 줄 요약: SmartFlip은 회원가입이 필요 없으며, 회사는 직접적으로 어떤 개인정보도 수집하지 않습니다. 단, 광고 표시 및 앱 안정성 개선을 위해 Google AdMob과 Firebase(Analytics, Crashlytics) 서비스가 익명 데이터를 처리합니다. 모든 학습 데이터는 사용자의 기기에만 저장됩니다.

제1조 (수집하는 개인정보 항목)

1. 회사가 직접 수집하지 않는 정보

회사는 다음 정보를 일체 직접 수집하지 않으며 자체 서버를 운영하지 않습니다.

  • 이름, 이메일, 전화번호, 생년월일 등 개인식별정보
  • 위치 정보(GPS)
  • 연락처, 통화 기록
  • 비밀번호, 결제 정보

2. 제3자 서비스(SDK)가 처리하는 정보

광고 표시 및 앱 안정성 개선을 위해 다음 Google 서비스가 익명/비식별 데이터를 처리합니다. 이 데이터는 회사 서버를 거치지 않고 Google에 직접 전송됩니다.

  • Google AdMob (광고): 광고 식별자(Android Advertising ID), 기기 모델/OS 버전, IP 주소, 광고 상호작용(노출/클릭) — 광고 노출 및 사기 방지
  • Firebase Analytics (분석): 앱 인스턴스 ID, 화면 뷰 이벤트, 사용자 속성(언어 등), 익명 사용 패턴 — 앱 개선용 통계
  • Firebase Crashlytics (크래시 보고): 크래시 스택 트레이스, 기기 모델/OS 버전, 앱 버전 — 버그 진단 및 안정성 개선 (release 빌드만 활성)

위 데이터는 모두 익명 또는 비식별 형태이며 사용자 본인을 직접 식별하는 데 사용되지 않습니다. 자세한 처리 내용은 다음 정책을 참조하세요:

  • Google 개인정보처리방침
  • Firebase 개인정보 정책
  • AdMob 광고 데이터 처리

2. 기기 내부에만 저장되는 데이터

다음 데이터는 사용자의 기기 저장소에만 보관되며, 외부 서버로 전송되지 않습니다.

항목 용도 저장 위치
학습 진도(SRS 데이터) 간격 반복 학습 알고리즘 로컬 SQLite
사용자가 만든 카드 학습 콘텐츠 로컬 SQLite

설정 정보(언어, 테마, 알고리즘 등) 앱 환경 설정 SharedPreferences

SmartFlip은 다음 권한을 다음 목적으로만 사용합니다.

제3조 (개인정보의 처리 목적)

회사는 자체 서버를 운영하지 않으며, 사용자의 데이터를 다음 목적으로만 기기 내에서 처리하거나 Google 서비스(AdMob, Firebase)로 익명 데이터를 전송합니다.

  1. 학습 기능 제공: 플래시카드 학습, 퀴즈, 간격 반복 학습 알고리즘
  2. 사용자 콘텐츠 관리: 사용자가 만든 카드의 저장, 검수, 학습
  3. 학습 통계 표시: 사용자 본인의 학습 진도와 성과 시각화
  4. 게이미피케이션: EXP, 레벨, 데일리 미션, 스트릭 뱃지
  5. 광고 표시 (Google AdMob): 무료 앱 운영을 위한 배너/전면/리워드 광고 노출
  6. 앱 안정성 개선 (Firebase Crashlytics): 크래시 자동 보고로 버그 진단
  7. 앱 사용 분석 (Firebase Analytics): 익명 사용 패턴 분석으로 UX 개선

제4조 (개인정보의 제3자 제공 및 처리위탁)

회사는 자체적으로 사용자의 개인정보를 제3자에게 제공하지 않습니다. 다만, 제1조 2항에 명시된 Google 서비스에 익명/비식별 데이터가 직접 전송되어 처리됩니다.

처리위탁 받는 자 (Google LLC)

위탁받는 자위탁 업무처리 데이터
Google LLC (AdMob) 광고 노출 및 사기 방지 광고 ID, 기기 정보, IP, 광고 상호작용
Google LLC (Firebase Analytics) 익명 사용 패턴 분석 앱 인스턴스 ID, 화면 뷰, 이벤트, 언어 설정
Google LLC (Firebase Crashlytics) 크래시 보고 스택 트레이스, 기기/OS/앱 버전

이용자는 기기 설정에서 광고 ID 재설정 또는 광고 개인 맞춤 설정 거부가 가능합니다 (Android: 설정 > Google > 광고 > 광고 ID 삭제).

제5조 (사용자가 선택적으로 공유하는 데이터)

다음은 사용자의 명시적 액션이 있을 때만 처리됩니다.

  • .smartflip 파일 공유: 사용자가 만든 카드 세트를 다른 사람과 공유할 때, 사용자가 직접 공유 메뉴(카카오톡, 이메일 등)를 통해 파일을 전송합니다. 이 파일에는 카드 내용만 포함되며, 학습 진도 등 개인 데이터는 포함되지 않습니다.
  • .sfbackup 파일 백업/복원: 사용자가 직접 백업 파일을 만들어 외부 저장소에 보관/복원할 수 있습니다. 회사는 이 과정에 관여하지 않습니다.
  • 피드백 이메일: 설정 > 문의/피드백을 통해 사용자가 직접 이메일을 작성해 보내는 경우, 사용자가 입력한 내용만 회사에 전달됩니다.
주의: .smartflip 또는 .sfbackup 파일을 공유하기 전, 카드에 개인정보(이름, 전화번호 등)가 포함되어 있지 않은지 확인해주세요. 파일이 외부로 나간 이후의 관리는 사용자의 책임입니다.

제6조 (개인정보의 보유 및 파기)

회사는 사용자의 데이터를 직접 저장하지 않습니다. 모든 데이터는 사용자의 기기에 저장되며, 다음 방법으로 삭제할 수 있습니다.

  • 앱 삭제: 모든 학습 데이터, 사용자 카드, 설정이 즉시 영구 삭제됩니다.
  • 앱 내 데이터 초기화: 설정 메뉴에서 카드, 학습 진도를 개별 삭제 가능합니다.
  • 휴지통 자동 정리: 삭제된 카드는 30일 후 자동 영구 삭제됩니다.

제7조 (이용자의 권리)

이용자는 다음과 같은 권리를 행사할 수 있습니다.

  1. 데이터 열람: 앱 내 통계 화면에서 본인의 모든 학습 데이터 확인 가능
  2. 데이터 수정/삭제: 앱 내 카드 관리 화면에서 직접 수정/삭제 가능
  3. 데이터 백업: .sfbackup 파일로 사용자 데이터 export 가능
  4. 전체 데이터 삭제: 앱 삭제로 모든 데이터를 즉시 삭제

제8조 (아동의 개인정보 보호)

SmartFlip은 만 13세 미만 아동의 개인정보를 의도적으로 수집하지 않습니다. 회사는 사용자 본인 식별 정보를 직접 수집하지 않으며, 광고는 Google AdMob의 가족 정책에 따라 부적절한 광고가 차단되도록 설정되어 있습니다.

제9조 (개인정보의 안전성 확보 조치)

  • 모든 학습 데이터는 기기 내부에만 저장 (회사 서버 없음)
  • 이미지 파일은 앱 전용 저장소에 보관 (다른 앱이 접근 불가)
  • 카드 입력 시 자동 sanitize (HTML/스크립트 제거, XSS 방지)
  • 민감정보 패턴 자동 감지 및 경고 (주민번호, 카드번호, SSN, 마이넘버 등)
  • 파일 공유 전 민감정보 경고 다이얼로그
  • 제3자 서비스(Google) 통신은 모두 HTTPS로 암호화
  • Crashlytics는 release 빌드에서만 활성 (개발/테스트 데이터 차단)

제10조 (개인정보처리방침의 변경)

Privacy Policy

SmartFlip (Smart Trivia) | Effective Date: May 7, 2026 | Version 1.1

RedMoon Studio ("Company", "we", "us") operates the SmartFlip mobile application. This Privacy Policy explains how we handle information when you use our app.

One-line Summary: SmartFlip does not require account registration, and we do not directly collect any personal information. However, Google AdMob and Firebase (Analytics, Crashlytics) process anonymous data for ad delivery and app stability. All learning data is stored only on your device.

1. Information We Collect

What We Do NOT Directly Collect

The Company does NOT directly collect any of the following and operates no servers:

  • Personal identifiers such as name, email, phone number, date of birth
  • Location data (GPS)
  • Contacts or call history
  • Passwords or payment information

Information Processed by Third-Party SDKs

The following Google services process anonymous/non-identifiable data for ad delivery and app stability. This data is sent directly to Google, not through Company servers.

  • Google AdMob (Advertising): Advertising ID (Android Advertising ID), device model/OS version, IP address, ad interactions (impressions/clicks) — for ad delivery and fraud prevention
  • Firebase Analytics: App instance ID, screen view events, user properties (locale, etc.), anonymous usage patterns — for app improvement statistics
  • Firebase Crashlytics: Crash stack traces, device model/OS version, app version — for bug diagnosis and stability improvement (release builds only)

All data above is anonymous or non-identifiable and is not used to directly identify you. For details, see:

  • Google Privacy Policy
  • Firebase Privacy and Security
  • AdMob Data Disclosure

Data Stored Only On Your Device

The following data is stored exclusively on your device's storage and is never transmitted to any external server:

Data Type Purpose Storage Location

User-created cards Learning content Local SQLite
User-attached card images Flashcard visual aids App-private storage
Quiz history, statistics, gamification data Learning motivation Local SQLite
Settings (language, theme, algorithm, etc.) App configuration SharedPreferences

  5. Ad Delivery (Google AdMob): Banner/interstitial/rewarded ads to support free app operation

  7. Usage Analytics (Firebase Analytics): Anonymous usage pattern analysis for UX improvement

The following data is processed only when you take explicit action:

第5条 (ユーザーが任意で共有するデータ)

以下はユーザーの明示的なアクションがある場合のみ処理されます。

注意: .smartflipまたは.sfbackupファイルを共有する前に、カードに個人情報(氏名、電話番号など)が含まれていないかご確認ください。ファイルが外部に出た後の管理はユーザーの責任となります。
